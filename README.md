# RMSN-Web

A high-performance, minimalist personal website and developer toolkit built with **Astro**, **Tailwind CSS**, and **KaTeX**. 

## Features

* **Hybrid Architecture:** Static Site Generation (SSG) for high-speed content delivery and Client-Side Rendering (CSR) for dynamic browser-based utilities.
* **Math Support:** Native LaTeX rendering.
    * **Static:** Pre-rendered via `remark-math` and `rehype-katex` for blog posts.
    * **Dynamic:** Client-side via `katex.min.js` and `auto-render` for the Reader utility.
* **Smart Syntax Highlighting:** Powered by Shiki with a custom UI header that adapts to language type or custom filenames.
* **Dark Mode:** Deep dark theme using `#0a0a0a` background and Plus Jakarta Sans/JetBrains Mono typography.
* **URL Hash Tools:** Stateless Reader and Encoder that utilize browser fragments (`#`) to bypass server-side redirect issues.
* **Privacy-First:** Integrated Simple Analytics with Do-Not-Track (DNT) support.

## Tech Stack

* **Framework:** [Astro 5.x](https://astro.build)
* **Styling:** [Tailwind CSS](https://tailwindcss.com) + Typography Plugin
* **Typography:** Plus Jakarta Sans & JetBrains Mono
* **Icons:** [Iconoir](https://iconoir.com)

## Logic & Implementation

### 1. Code Block Headers
The site uses a custom client-side script that parses the data-language attribute generated by Shiki during build time:
- Detected Language: Displays the uppercase name (e.g., PYTHON).
- Custom Name: If a filename is provided in the backticks (e.g., ```main.rs), the header displays that string.
- Empty/Plaintext: Hides the header label entirely for a cleaner aesthetic.

### 2. Stateless Utilities (Reader & Encoder)
To ensure reliability across all hosting environments (avoiding trailing-slash redirects), the Reader uses the URL Fragment Identifier (#) instead of query parameters:
- Encoding: Uses TextEncoder + btoa + Base64URL normalization.
- Decoding: Uses atob + TextDecoder to handle Unicode characters safely, replacing deprecated escape/unescape methods.
- URL Guard: The Encoder features a real-time safety check to warn if the payload exceeds the ~2000 character browser limit.

### 3. Navigation Hierarchy
- Desktop: Features a hover-activated dropdown for "Tools."
- Mobile: Implements a full-screen drawer menu with a transition-safe "hamburger" toggle and backdrop-blur styling.
- Active State: Navigation links use path normalization to ensure active states remain highlighted regardless of trailing slashes.
